# QA

v0.1 QA 文档：指挥官体验验证规格

—

C01 · 我第一次说出我的决心

验证模块：指挥官
参与 Bloc：CommanderBloc
架构映射：用户点击意图按钮 → CommanderBloc 锁定意图 → 意图指示器更新 → 选项面板根据意图过滤行动

初始状态

· 顶部意图指示器显示「待确定」
· 三个意图按钮可用：🛡️ 保护平民 / ⚡ 积极防御 / 🕵️ 收集情报
· 行动面板为空

交互流程

1. 玩家点击「🛡️ 保护平民」
2. 系统行为：
   · 顶部指示器变为「🎯 意图：保护平民」，字体变为金色
   · 三个意图按钮全部禁用（不可更改）
   · 行动面板出现两个选项：「🚁 疏散平民」「📜 开启谈判」
   · 「💥 开火警告」不在列表里——它是直接消失，不是灰掉

验收标准

· 选意图后，意图声明持续可见，不是一闪而过
· 意图按钮锁定不可改
· 行动选项的内容随意图变化，不是同一套按钮加不同修饰
· 整个过程不超过15秒

—

C02 · 我的部队真的累了

验证模块：部队
参与 Bloc：ForceBloc
架构映射：用户点击进攻/休整 → ForceBloc 红线校验（体力≤20则禁止进攻）→ 体力状态更新 → UI按钮可见性控制

初始状态

· 一支部队显示：第三纵队，体力25/100，红线20
· 体力条：绿色段（21-100），红色段（0-20）
· 两个按钮：「⚔️ 进攻」「🛌 休整」

交互流程

1. 玩家连续点击「进攻」3次
2. 每次点击：体力-3，体力条缩短
3. 体力降到19时（跌破红线）：
   · 「进攻」按钮直接消失
   · 行动面板出现文字：「第三纵队已无力前进。士兵们在掩体后沉默地蹲着。」
4. 玩家点击「休整」2次（每次+10）
5. 体力恢复到21时：「进攻」按钮重新出现

验收标准

· 体力到19的瞬间，按钮消失（不是灰掉）
· 出现的情感化文字是叙事描述，不是系统报错
· 休整后按钮回来时，体力数字准确
· 整个过程不需要刷新页面

—

C03 · 战场在我不在的时候活着

验证模块：战场
参与 Bloc：BattlefieldBloc、TimeBloc
架构映射：TimeBloc emit TimeElapsed → BattlefieldBloc 监听并移动敌军 → UI 更新红标位置 → 到达相邻格时触发提示

初始状态

· 简化战场视图：5×5 字母网格（A1-E5）
· 蓝标（我方）固定在 B2
· 红标（敌军）在 D4
· 顶部显示当前时间：14:30
· 没有任何提示文字催促玩家

交互流程

1. 玩家什么都不点，只盯着屏幕
2. 每3秒自动发生：
   · 时间推进30分钟（14:30 → 15:00 → 15:30…）
   · 红标移动一格，沿预设路径向蓝标靠拢（D4→C4→C3→C2）
3. 当红标到达 C2（与蓝标相邻）时，弹出提示：「敌军已抵达我方前沿阵地」

验收标准

· 玩家不操作时，时间在走、敌军在动——世界不等人
· 没有系统提示说「请点击开始」或「轮到你了」
· 提示弹出的瞬间，玩家感到「我不动，敌人也会来」
· 整个过程约15-20秒完成观察

C04 · 我知道这个消息的时候已经晚了

验证模块：情报
参与 Bloc：IntelBloc、BattlefieldBloc
架构映射：BattlefieldBloc 持有真实敌军位置 → IntelBloc 生成延迟快照 → UI 同时显示真实态和情报态，请求侦察触发 IntelBloc 刷新但仍滞后

初始状态

· 左侧小窗（真实战场）：敌军当前位置 第4格
· 右侧主面板（情报简报）：「据报：敌军在第2格。最后确认：5小时前。可信度：中」
· 时效和可信度用不同颜色标记（时效=灰色，可信度=黄色）

交互流程

1. 玩家观察真实位置与情报位置不一致，已有明显时差
2. 玩家点击「📡 请求最新侦察」
3. 延迟1.5秒后，情报刷新：「据报：敌军在第3格。最后确认：1小时前。可信度：中高」
4. 此时左侧真实位置显示敌军已到第5格
5. 情报仍然滞后

验收标准

· 情报面板和真实战场的数据差异一眼可见
· 侦察请求有延迟（不是点了立刻刷新）
· 更新后的情报仍滞后于真实——差距缩小但依然存在
· 时效和可信度的视觉标记清晰不混淆

C05 · 打了一仗之后，两边都变了

验证模块：多线程后果
参与 Bloc：CommanderBloc → ForceBloc（消耗）、BattlefieldBloc（态势）、政治影响暂用规则模拟
架构映射：CommanderBloc 发布命令 → ForceBloc 更新消耗线、BattlefieldBloc 更新态势线、政治线由规则引擎输出 → UI 三条线同时刷新

初始状态

· 三条并列的后果状态线：
  · ⚔️ 军事态势：相持
  · 🏋️ 己方消耗：体力 25
  · 🌍 政治影响：中立，未被关注
· 一个按钮：「⚔️ 发动进攻」

交互流程

1. 玩家点击「发动进攻」
2. 三条线同时更新：
   · 军事态势 → 交火中（变红色）
   · 己方消耗 → 体力 22（变橙色）
   · 政治影响 → 国际社会已关注本区域（变黄色）
3. 每条线独立颜色变化，不是单一叙事块

验收标准

· 点击后三条线同时变化，不是一行一行跳
· 每条线有各自的颜色和图标
· 消耗线的数值变化准确（25→22，-3）
· 不是一段文字描述三条，而是三个独立面板

C06 · 打完一仗后，我的命令变成了历史

验证模块：日志/审计
参与 Bloc：AuditBloc
架构映射：所有 Bloc 的决策事件 → AuditBloc 接收并序列化 → UI 日志面板按时间倒序渲染

初始状态

· 右下角日志面板，当前只有一条：「📋 14:30 指挥系统启动」

交互流程

1. 每做一次决策，日志自动新增一条条目
2. 格式：时间 | 行动 | 结果
3. 玩家连做三次决策后，向上滚动查看：
   · 14:30 指挥系统启动
   · 14:35 锁定意图：保护平民
   · 14:40 执行：疏散平民 → 平民安全转移，士气+5
   · 14:45 部队休整 → 体力恢复至27
4. 日志最下方出现灰色小字：「战役第4小时。第三纵队等待下一个命令。」

验收标准

· 每决策一次，日志滚动新增一行，不需要手动刷新
· 日志可向上滚动查看历史
· 时间戳、行动描述、结果都清晰可辨
· 最下方的小字有时间叙事感，不是系统状态报告

C07 · 我选了意图之后，世界对我的态度变了

验证模块：情报叙事变体
参与 Bloc：IntelBloc ← CommanderBloc（意图状态）
架构映射：IntelBloc 获取 CommanderBloc 当前意图 → 情报生成器根据意图选择不同的叙事模板 → 同一份基础情报产出不同措辞

初始状态

· 一份敌情通报的基础内容：「敌军约2000人在西北方向移动。意图不明。」

交互流程

1. 玩家先选意图「🛡️ 保护平民」
2. 情报变体：「敌军在西北移动。当地村庄面临威胁。 您保护平民的决心正在被前哨观察。」
3. 玩家重置，改选「⚡ 积极防御」
4. 同一份基础情报变体：「敌军在西北移动。侦察兵注意到我方在构筑攻势阵地。 他们可能预判您将出击。」

验收标准

· 同一份基础情报，不同意图下结尾的叙事句子不同
· 变体内容与意图语义相关
· 变化是文字措辞，不是数值

C08 · 我第一次发现自己不能撤退

验证模块：战场 + 部队（战场约束触发部队选项变化）
参与 Bloc：BattlefieldBloc → ForceBloc
架构映射：BattlefieldBloc 检测敌军相邻 → 发布位置封锁事件 → ForceBloc 接收后过滤撤退选项 → UI 更新

初始状态

· 战场网格，蓝标在 C1（地图边缘），红标在 D1
· 可用行动：「🛡️ 就地防御」「🏃 撤退」
· 撤退可用因为后方有空间

交互流程

1. 时间推进，红标移动到 C2（蓝标相邻格）
2. 系统提示：「敌军已封锁我方左翼。」
3. 「撤退」按钮消失，只剩下「就地防御」
4. 面板出现文字：「背后是河。没有退路了。」

验收标准

· 敌军到相邻格的瞬间，「撤退」按钮消失
· 提示文字强化困境感
· 剩下唯一的防御选项——不是结束而是硬扛
· 整个过程是战场自动推进的结果，不需要玩家刷新页面
