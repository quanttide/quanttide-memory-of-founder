# 用户故事地图

对用户故事地图方法的新认知

—

What： 叙事先于逻辑

传统做法是先定义角色、功能、流程，再用用户故事去填充。我们的新认知是反过来的：先讲故事，在故事中自然捕捉需求，再提炼成用户故事卡片。

逻辑推导容易漏掉那些“说不清但很重要”的东西——犹豫、踏实、感觉不对。叙事能让这些东西浮现。需求不是被推导出来的，是在故事中被发现的。

Who： 组织法人是合法的叙事主体

用户故事地图传统上以“用户”为中心，但用户不一定是某个具体的自然人。组织也可以被人格化为一个活生生的叙事主体。

“我是量潮。我看到陈小明在华为项目上犹豫了。我感觉到红点亮起来。我做了决定。我安静下来。”这个叙事视角让信息流转、层级判断、决策闭环这些抽象概念变成了可以被感知、被讲述的经验。

组织法人不是替代具体角色，而是提供顶层视角，让所有角色的碎片在一个共同的叙事容器里找到位置。

How：从碎片中创造全景图

人不善于讲完整故事，但人善于产生碎片化的需求、痛点、想法。

要求每个角色“讲你的一天”，门槛太高。但问“你想在看板的什么地方，能让你更快做判断？”碎片就出来了。

用户故事地图不要求碎片被预先整理成故事，而是提供一种方法，让碎片在结构中自然碰撞出关联，涌现出全景。

感知、理解、决定、行动——这四个阶段不是管理术语，是碎片分类器。

任何角色在任何时刻的碎片，都可以被归入这四个格子之一。同一件事在不同角色之间的碎片，被分类后，关联会自动浮现——感知层的碎片连着理解层的碎片，理解层的连着决定层的。

OODA在这里不是指挥人的循环，是组织碎片的骨架。


从碎片到全景，是一个创造环，不是一次分析。

收集碎片 → 按OODA分类 → 连成链 → 法人视角串讲。念通了，全景图就出来了。念不通，说明某个环节碎片不够，回去收集。

全景图不是一次性画出来的，是在这个循环中一圈一圈转出来的。每一圈都让秩序更清晰。


PDCA是控制环——有标准，消除偏差。OODA是适应环——有变化，比变化更快。

我们从碎片到全景的方法，是第三种循环：创造环——有碎片，放入结构，碰出关联，生成全景。它不修正，不追赶，它让秩序从碎片中涌现。

这就是我们对用户故事地图方法的核心新认知：它不是一份文档格式，不是一个需求模板，而是一个让组织从碎片中共同创造出共享全景图的创造环。
