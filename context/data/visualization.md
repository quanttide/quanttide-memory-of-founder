# 可视化

你描述的这个“大方块套小方块，小方块里还有东西”的画面，其实已经非常接近一种专门展示层次数据的可视化方法了。它已经不完全是普通的二维散点或柱状图，而是要在二维平面上表现包含关系和层级结构。

针对这种场景，最贴切的方法主要有以下几种：

1. 矩形树图 (Treemap)

这就是你脑海中的那个方法：用一个最大的矩形代表整体，然后在里面切分成更小的矩形代表各个部分，小矩形里面还能继续嵌套。

· 适合什么：数据有明确的数值权重（如文件大小、销售额、预算占比）。面积越大代表数值越大，颜色可以代表另一个维度（如增长率）。
· 典型例子：磁盘空间分析工具，一眼看出哪个文件夹占空间最大。
· 优点：空间利用率极高，擅长展现相对大小。
· 局限：如果层级太深或矩形太小，细节就看不见了，也不适合表现不强调大小的纯粹层级。

2. 树图 / 节点链接图 (Tree / Node-link Diagram)

如果“包含关系”本身比“面积大小”更重要，就用连线和方块来表示。

· 形式：用方块代表每个节点，用线条把“父方块”和“子方块”连接起来。可以做成从左到右或从上到下的组织结构图 (Org Chart)。
· 适合什么：家族谱、公司组织架构、分类学——重点在看清脉络和父子关系，而不是谁占的地盘大。
· 优点：层级关系一目了然，清晰直观。
· 局限：节点多了会显得很乱，占用空间大。

3. 冰柱图 (Icicle Chart)

这个像是一个“横着摆的柱状图层级版”。根节点在最上面，下面一层层地划分出不同宽度的细长矩形，看起来像挂在屋檐下的冰柱。

· 适合什么：想看层级结构的同时，也想看清每个层级内部各部分的比例关系。
· 优点：能同时展示层数和同级间的比例，比例靠宽度来体现。

4. 缩进树 (Indented Tree)

最简单直接，类似电脑里的文件管理器：通过列表缩进来表示层级的深浅，前面的小三角可以展开收起。

· 适合什么：数据量大、层级深，主要为了查找和浏览，不要求一眼看全貌。这是最实用的信息架构。

—

简单来说，怎么选：

· 强调各部分的“大小占比”，想一眼找到最大的那几块 → 优先用 矩形树图。
· 强调“谁归谁管”的层级路径，像看家谱一样 → 用 组织结构图式树图。
· 既要看层级，也要看同级内的比例 → 可考虑 冰柱图。
· 纯粹为了查阅和操作 → 文件列表式的 缩进树。

你说的那个场景，很可能就是矩形树图。如果给它加上交互（可以点击大矩形放大进入下一层），就是最常见的层级数据探索工具了。
